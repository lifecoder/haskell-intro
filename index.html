So we will start by defining what is functional programming. And what is Haskell and how it is different from other languages. We will quickly look at a few most important language features. Actually Haskell is a very deep language and there is so much more to it than what I will tell you today. You would not learn Haskell today but I want to give you a general impression of what it is and what problems it can solve. I wont go through its syntax because I hope we all can agree that learning a syntax of a new language is the most quick and easy part for every software engineer. If you get interested in it you can learn it yourself by using freely available online books that will give you enough information to start writing simple programs after few evenings. After this I will give you an overview of its platform and runtime. I will also tell you about few libraries that I consider to be very elegant and different from everything I've used in imperative languages. I also would like you to hold your questions until the end because of time constraints.

So lets start... Functional programming is sometimes made fun of because of so called "absence of side effects". By side effects I mean mutation, input output and anything that is not pure computation. So the language without side effects is useless. Of course you can have side effects in major functional programming language like Scala, Haskell, Erlang, Scheme, Clojure and others.

Why you would want to learn it?

  • It is the coolest language ever (sorry lispers...)
  • New way of thinking
  • New patterns and ideas
  • Very expressive and concise
  • Opens whole new class of languages/tools
So why would you want to learn it? I mean .. learning a new programming language takes some time. And learning a completely different paradigm takes a LOT of time. For me personally it was very rewarding experience. I gave me a really new perspective on thinking about programming. And by "new perspective" I mean really new. Because, and I think you know this, that after few static and a few dynamic imperative languages that you learn you start to see that they are very similar and they have same approach for solving problems. Only thing that is different is a community, ecosystem of libraries, and little tiny details that some languages emphasize and others don't. And this forms a programming style in Ruby and makes it different from for example Python. Learning a new imperative language is boring... Learning a language from a different paradigm opens whole new class of patterns and ideas. It is also very expressive and concise... You will look at your few lines of code and think that those are the most amazing few lines of code you've written. And we always talk about choosing a best tool for the job. But learning a functional language opens a whole new class of tools for you. You wont be afraid to jump in and rewrite some web service in Erlang to make it more fault tolerant and concurrent. You would not be afraid to use Scala for Android applications that you write in Java. And you hate java... and you cant do anything about it if you dont know if you can program in functional languages... thats what I did on my previous job.

What is Functional Programming?

  • Function application
  • Higher order functions
  • Avoids state and mutable data
  • Recursion
  • Pure functions over side effects
  • Favour composition
  • Type systems

What is Haskell?

  • Purely-functional programming language
  • Lazy
  • Statically typed
  • Type inference
  • Strongly typed
  • Parametric polymorphism

Purely-functional

  • Everything is immutable (almost)
  • Functions are values
  • No Side effects (almost)
  • Recursion, TCO
